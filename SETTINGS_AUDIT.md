## 2. Preference Keys Audit

- **No Dead Settings**: Every setting option presented in the user interface binds to a valid `bharadwajsanket.aether.music.constants.PreferenceKeys` property.
- **No Orphan Preferences**: Keys defined in `PreferenceKeys.kt` are actively resolved either directly by Compose screens, API engines, or media services.
- **No Duplicated Keys**: All Datastore name definitions are unique. Shared states (such as `MixSortDescendingKey` and `AlbumSortDescendingKey` referencing `"albumSortDescending"`) are intentional sharing of configuration states.
